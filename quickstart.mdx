## Get started with Bool CLI

This guide will take you from installation to your first successful deployment in just a few steps.

<Steps>
  <Step title="Install Bool CLI">
    Install the CLI globally using npm, yarn, or pnpm:

    <CodeGroup>

    ```bash npm
    npm install -g bool-cli
    ```

    ```bash yarn
    yarn global add bool-cli
    ```

    ```bash pnpm
    pnpm add -g bool-cli
    ```

    </CodeGroup>

    Verify the installation:

    ```bash
    bool --version
    ```
  </Step>

  <Step title="Quick deploy with shipit (no auth required)">
    The fastest way to get code live—no API key needed:

    ```bash
    bool shipit
    ```

    This creates an anonymous Bool and deploys all files from your current directory. You'll see output like:

    ```
    https://zx9k2m.bool01.com
    ✔ Shipped "my-project" (12 files)
      https://zx9k2m.bool01.com
      slug: zx9k2m
    ```

    <Note>
    Your project is now live! The CLI automatically creates a `.bool/config` file to track this Bool for future updates.
    </Note>

    To update the same Bool, just run `bool shipit` again from the same directory:

    ```bash
    bool shipit --message "Update homepage"
    ```

    Output:

    ```
    https://zx9k2m.bool01.com
    ✔ Updated v2 (12 files)
      https://zx9k2m.bool01.com
      slug: zx9k2m
    ```
  </Step>

  <Step title="Authenticate for full features (optional)">
    For authenticated deployments and project management, log in with your API key:

    ```bash
    bool auth login
    ```

    You'll be prompted to enter your API key:

    ```
    Enter your bool.com API key: 
    ```

    <Note>
    Get your API key from the [bool.com](https://bool.com) web UI in your account settings.
    </Note>

    After entering your key, you'll see:

    ```
    API key saved to ~/.config/bool-cli/config.json
    ```

    Verify your authentication:

    ```bash
    bool auth status
    ```

    Output:

    ```
    API key: sk_live_…abc123
    API is reachable and healthy.
    ```

    <Warning>
    Keep your API key secure. You can also set it using the `BOOL_API_KEY` environment variable instead of storing it in a config file.
    </Warning>
  </Step>

  <Step title="Create and deploy a project">
    With authentication set up, you can create named Bools and deploy with full version control:

    ```bash
    bool create my-awesome-app
    ```

    Output:

    ```
    Created "my-awesome-app" (my-awesome-app)
    URL: https://my-awesome-app.bool01.com
    ```

    Deploy files to your Bool:

    ```bash
    bool deploy my-awesome-app --message "Initial deployment"
    ```

    Output:

    ```
    Deploying 15 file(s) from /home/user/my-awesome-app…
    Deployed v1 (15 files)
    Live URL: https://my-awesome-app.bool01.com
    ```
  </Step>

  <Step title="List your projects">
    View all your Bools:

    ```bash
    bool list
    ```

    Example output:

    ```
    ┌─────────────────┬─────────────────┬────────────┬──────────────────────────────┬─────────────────────┐
    │ Name            │ Slug            │ Visibility │ URL                          │ Updated             │
    ├─────────────────┼─────────────────┼────────────┼──────────────────────────────┼─────────────────────┤
    │ my-awesome-app  │ my-awesome-app  │ private    │ https://my-awesome-app.b...  │ 2026-03-03T10:30:00 │
    │ landing-page    │ landing-page    │ public     │ https://landing-page.boo...  │ 2026-03-02T15:20:00 │
    └─────────────────┴─────────────────┴────────────┴──────────────────────────────┴─────────────────────┘
    ```

    <Note>
    Use `--limit` to control how many Bools are shown (default is 20):
    ```bash
    bool list --limit 50
    ```
    </Note>
  </Step>

  <Step title="View project details">
    Get detailed information about a Bool:

    ```bash
    bool show my-awesome-app
    ```

    Output:

    ```
    Name: my-awesome-app
    Slug: my-awesome-app
    Visibility: private
    Description: (none)
    URL: https://my-awesome-app.bool01.com
    Created: 2026-03-03T10:30:00
    Updated: 2026-03-03T10:30:00

    Latest version: v1
      Files: 15
      Message: Initial deployment
      Created: 2026-03-03T10:30:00
    ```

    If you're in a directory with a `.bool/config` file, you can omit the slug:

    ```bash
    cd my-awesome-app
    bool show
    ```
  </Step>
</Steps>

## Working with versions

View deployment history:

```bash
bool versions my-awesome-app
```

Example output:

```
┌─────────┬───────┬─────────────────────┬─────────────────────┐
│ Version │ Files │ Message             │ Created             │
├─────────┼───────┼─────────────────────┼─────────────────────┤
│ v3      │ 15    │ Add dark mode       │ 2026-03-03T14:20:00 │
│ v2      │ 15    │ Update homepage     │ 2026-03-03T12:15:00 │
│ v1      │ 15    │ Initial deployment  │ 2026-03-03T10:30:00 │
└─────────┴───────┴─────────────────────┴─────────────────────┘
```

Pull a specific version:

```bash
bool pull my-awesome-app ./backup --version 2
```

## Smart file handling

Bool CLI automatically respects `.boolignore` files (using gitignore syntax):

```text .boolignore
node_modules
.env
*.log
dist
.DS_Store
```

Default exclusions include `.git`, `node_modules`, `__pycache__`, `.DS_Store`, and `.bool`.

You can also exclude files on the fly:

```bash
bool deploy --exclude "*.test.js" --exclude "__mocks__"
```

## JSON output

All commands support `--json` for machine-readable output:

```bash
bool list --json
bool show my-awesome-app --json
bool versions my-awesome-app --json
```
