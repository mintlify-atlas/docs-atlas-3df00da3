## Common patterns

### JSON output

Most commands support a `--json` flag for machine-readable output:

```bash
bool list --json
bool show my-site --json
bool versions my-site --json
```

### Slug resolution

Many commands accept an optional `[slug]` argument. If omitted, the CLI reads the slug from `.bool/config` in the current directory:

```bash
# Explicit slug
bool show my-site
bool deploy my-site

# Implicit slug (reads from .bool/config)
cd my-project
bool show
bool deploy
```

The `.bool/config` file is automatically created and updated when you create, deploy, or pull a Bool.

### Exit codes

Commands exit with code `0` on success and `1` on error.
